CanvasWrapper

The CanvasWrapper takes a canvas from the DOM and renders a tree of Widgets on it.

Beside rendering Widgets into the Canvas, it also converts user input like native mouse events into Wings-understandable “internal” events.  Example: The user moves the mouse over a Widget, (or a subclass of it), the CanvasWrapper converts this interaction into a native_mouseMove event, which is then processed by the Widget itself.  For more in-depth information, have a look at the “Canvas.*” event handlers.

_self

Keeps an internal reference on “this” for simplifying access to it.

_canvas

The Canvas DOM Element which this CanvasWrapper is wrapped around.

_ctx

This is the graphical context of _canvas.  Use it to draw stuff on the Canvas DOM Element.

_boostWidgets

This is a simple associative array which holds all Widgets currently available somewhere in the widget hierarchy.  By keeping this list, the search algorithms for specific Widgets can be implemented iterative instead of recursive to improve the overall performance.

See also

  • prepareSearchBoost()

_boostWidgetIds

In addtion to _boostWidgets, this Array keeps track of each Widgets ID.  The Array itself is sorted to the corresponding depth in the widget hierarchy from the deepest to the highest.  This makes a major improvment to <searchDeepestWidgetOnPosition()>s searching performance.

createDefaultStrategy()

Creates a DefaultStrategy and attaches a “defaultstrategy:startdrag” event listener to it.  As soon as the event gets detected, the CanvasWrappers mouse strategy gets replaced with the result of createDragAndDropStrategy().
